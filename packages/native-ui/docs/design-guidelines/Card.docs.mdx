## What is a Card?

A Card is a self-contained unit of information or content. It serves as a container and includes various design presets.

A card can contain various elements such as text, images, buttons, and other user interface components like lists. It
is typically used to display related information in a visually consistent and organised way.

While the general structure and style of a card are often defined by the design system, the specific content within each
card can vary. There are no predefined design elements for the Card's content, except for instances where a card is nested
within another Card.

The Card doesn't have an inherent state but can contain elements with state-like Buttons or Icon Buttons.

## Card style variants

To clarify what a variant is, [please consult the documentation](https://wiki.uw.systems/posts/design-systems-glossary-ama05dma#hxsg1-component-variants).

Cards come in a variety of style variants for different content and contexts.

## colorScheme

To clarify what a colorScheme is, [please consult the documentation](https://wiki.uw.systems/posts/naming-conventions-and-stack-mirroring-z1olqysr).

The Card colorScheme determines the background and border colour of the Cards, not its content.

Cards come in a variety of colour schemes that can be combined with variants, padding, etc.

**base** uses white in light mode and dark grey in dark mode.

**Example for maintenance:** If a design requires a different background and border colour for the Card component (and is not an exception), a new ColorScheme should be added.

## Surface

The surface colour refers to the background colour of a specific element or area within UI.
A Card can have different variants of colour schemes in addition to the surface colour.

By selecting the right surface colour and card type, you can improve the overall content
hierarchy of your composition

**Technical considerations:**

- purple surface should be used only on light mode, in dark mode it looks the same as base.
- base is the default surface.

**Theming:** The Card uses Figma modes to manage dark and light themes. [Please refer to the
Figma documentation for more details on this functionality](https://help.figma.com/hc/en-us/articles/15343816063383-Modes-for-variables). Light is the default mode.

**Example for maintenance:** If a design requires a different surface color for any Card component,
a new surface should be added for both themes.

## Nested Card

When cards are placed within one another, we modify the border radius of the inner card
to enhance the visual appeal and create a cohesive layout. This can be accomplished by
using the nested Card within Card.

The nested Card has the same properties as Card.

<DocComponentWrap>
  <VStack space="xl">
    <Card padding="large">
      <HStack space="lg">
        <Card padding="large" variant="outline" style={{ flex: 1 }} nested>
          <Text style={{ marginBottom: 16 }}>Nested Card</Text>
        </Card>
        <Card padding="large" variant="outline" colorScheme="grey" style={{ flex: 1 }} nested>
          <Text style={{ marginBottom: 16 }}>Nested Card</Text>
        </Card>
        <Card padding="large" variant="outline" colorScheme="purple" style={{ flex: 1 }} nested>
          <Text style={{ marginBottom: 16 }}>Nested Card</Text>
        </Card>
      </HStack>
    </Card>
    <Card padding="large">
      <HStack space="lg">
        <Card padding="large" variant="outline" nested>
          <Center p="$5">
            <Text>Nested Card</Text>
          </Center>
        </Card>
      </HStack>
    </Card>
    <Card padding="large">
      <HStack space="lg">
        <Card
          padding="large"
          variant="outline"
          colorScheme="purple"
          style={{ width: '100%' }}
          nested
        >
          <Center p="$5">
            <Text>Nested Card</Text>
          </Center>
        </Card>
      </HStack>
    </Card>
  </VStack>
</DocComponentWrap>

## Padding

Padding is a property that adjusts the card's padding from 16 to 0, maintaining consistent usage.

**Use the props to adjust the padding and not overrides.**

<DocComponentWrap>
  <Card padding="medium">
    <HStack space="lg">
      <Card padding="large" variant="outline" nested>
        <Text style={{ marginBottom: 16 }}>Nested Card</Text>
      </Card>
      <Card padding="large" variant="outline" colorScheme="grey" nested>
        <Text style={{ marginBottom: 16 }}>Nested Card</Text>
      </Card>
      <Card padding="large" variant="outline" colorScheme="purple" nested>
        <Text style={{ marginBottom: 16 }}>Nested Card</Text>
      </Card>
    </HStack>
  </Card>
</DocComponentWrap>

_The padding is set to medium_

## Customisation and overrides

- Avoid overriding Card design presets
- Consistency: Overriding presets can create UI inconsistencies.
- Maintainability: Using the design system component simplifies updates during redesign.
- For new card styles that existing options don't cover, consult the design system team.
