## Accessibility

The `Grid` component is an all purpose component. By default, it has no
accessibility concerns. If you use the `Grid` as a custom element, it is up to
you to manage the resulting accessibility implications.

## Breakpoint Props

Instead of the stock MUI breakpoint prop names, the Web UI grid has props which
reflect our custom breakpoint names: {Object.keys(breakpoints).join(', ')}

## Guidelines

The `Grid` component should be used for [macro layouts](https://web.dev/learn/design/macro-layouts/),
and shouldn't be used inside other layout components. Due to the current implementation of the
`Stack` component, using the [lobotomized owl selector](https://alistapart.com/article/axiomatic-css-and-lobotomized-owls/),
you may find that rendering a `Grid` as a child of `Stack` breaks the layout.
Please avoid this.
